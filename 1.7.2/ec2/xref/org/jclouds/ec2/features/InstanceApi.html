150     * number specified for each AMI and allocate the remaining available
151     * instances using round robin.
152     * <p/>
153     * <h4>Security Groups</h4>
154     * <b>Note:</b> Every instance is launched in a security group (created using
155     * the CreateSecurityGroup operation.
156     * <h4>Key Pair</h4>
157     * You can provide an optional key pair ID for each image in the launch
158     * request (created using the CreateKeyPair operation). All instances that
159     * are created from images that use this key pair will have access to the
160     * associated public key at boot. You can use this key to provide secure
161     * access to an instance of an image on a per-instance basis. Amazon EC2
162     * public images use this feature to provide secure access without passwords.
163     * <p/>
164     * <b>Note:</b> Launching public images without a key pair ID will leave them
165     * inaccessible.
166     * <p/>
167     * The public key material is made available to the instance at boot time by
168     * placing it in the openssh_id.pub file on a logical device that is exposed
169     * to the instance as /dev/sda2 (the instance store). The format of this file
170     * is suitable for use as an entry within ~/.ssh/authorized_keys (the OpenSSH
171     * format). This can be done at boot (e.g., as part of rc.local) allowing for
172     * secure access without passwords.
173     * <h4>User Data</h4>
174     * Optional user data can be provided in the launch request. All instances
175     * that collectively comprise the launch request have access to this data.
176     * For more information, go the Amazon Elastic Compute Cloud Developer Guide.
177     * <h4>Product Codes</h4>
178     * 
179     * <b>Note:</b> If any of the AMIs have a product code attached for which the
180     * user has not subscribed, the RunInstances call will fail.
181     * <h4>Kernel</h4>
182     * 
183     * <b>Important:</b> We strongly recommend using the 2.6.18 Xen stock kernel
184     * with High-CPU and High-Memory instances. Although the default Amazon EC2

200     *           Amazon EC2 can launch, no instances a re launched at all.
201     *           Constraints: Between 1 and the maximum number allowed for your
202     *           account (default: 20).
203     * @param maxCount
204     *           Maximum number of instances to launch. If the value is more than
205     *           Amazon EC2 can launch, the largest possible number above
206     *           minCount will be launched instead. Constraints: Between 1 and
207     *           the maximum number allowed for your account (default: 20).
208     * @see #describeInstancesInRegion
209     * @see #terminateInstancesInRegion
210     * @see #authorizeSecurityGroupIngressInRegion
